## 2. 数据隔离检查

### 2.1 Class Service (class.service.ts)

- verifyTeacherAccess: ❌ 不存在
- teacherId 过滤: ❌ 不存在
- findAll() 方法直接查询所有班级，不根据当前登录教师过滤
- 前端 classes.js 调用 `GET /classes` 时会获取系统中所有班级
- 风险: Teacher 角色可以看到 Admin 创建的所有班级，包括未分配给自己的

### 2.2 Lesson Service (lesson.service.ts)

- verifyTeacherAccess: ❌ 不存在
- teacherId 过滤: ❌ 不存在
- findByClassCode() 直接返回指定班级的所有课时
- 风险: 任何 Teacher 可以查看任意班级的课时记录

### 2.3 Lesson Attendance Service (lesson-attendance.service.ts)

- verifyTeacherAccess: ❌ 不存在
- teacherId 过滤: ❌ 不存在
- batchRollCall() 不验证操作教师是否属于该班级
- 风险: 任何 Teacher 可以为任意班级记录考勤

### 2.4 Teacher Dashboard (teacher-dashboard.controller.ts)

- ✅ 有 teacherId 过滤
- getDashboard() 通过 TeacherAssignmentEntity 查询当前教师的分配班级
- 只统计自己分配的班级数据
- 这是唯一有数据隔离的 Teacher 相关接口

### 2.5 数据隔离总结

- 有隔离: Teacher Dashboard ✅
- 无隔离: Class List / Lesson List / Attendance Record ❌
- 根因: Class/Lesson/Attendance 的 Controller 使用 @Roles 控制访问权限，但不传递 teacherId 到 Service 层
- 影响范围: MVP 阶段可接受（教师数量少），规模化后需要修复

## 3. 发现的问题

### ISSUE-001: profile.js 课时响应格式不匹配
- Severity: P1
- Location: miniapp/pages/teacher/profile.js, loadRecentLessons() 函数
- 问题: 前端期望 `data.items`（分页格式），但后端 `GET /classes/:code/lessons` 返回的是扁平数组 `LessonEntity[]`
- 影响: 教师个人中心的"最近课程"功能永远显示空列表
- 代码: `if (!data || !data.items || data.items.length === 0) return null;`
- 修复: 改为 `var items = Array.isArray(data) ? data : (data && data.items ? data.items : []);`

### ISSUE-002: 教师端无数据隔离（Class/Lesson/Attendance）
- Severity: P2 (MVP阶段)
- Location: backend/src/modules/teaching/class/class.service.ts, lesson.service.ts, lesson-attendance.service.ts
- 问题: Teacher 角色可以访问所有班级/课时/考勤数据，不限于自己分配的班级
- 影响: 数据越权风险
- 建议: 在 ClassController.findAll() 中，当角色为 Teacher 时，通过 TeacherAssignment 过滤只返回该教师的班级
- 备注: 需要 Owner 决策是否在 MVP 阶段修复
