# EduERP-V4 v0.1 MVP Milestone
## 2026-07-20

## 版本概述

EduERP-V4 第一个双角色闭环版本。教师端完成课时录入流程，学生端完成课时查询流程。

## 已实现功能

### 教师端（backend + miniapp）
- 登录/认证（JWT + bcrypt）
- 班级列表查看（GET /api/v1/classes）
- 班级学生查看（GET /api/v1/classes/:code/students）
- 课时录入（POST /api/v1/lessons，含考勤状态 + 自动递增 lessonNumber）
- 考勤状态切换确认（带原因弹窗）
- 课时状态流转：DRAFT -> CONFIRMED -> COMPLETED -> CANCELLED

### 学生/家长端（backend + miniapp）
- 登录/认证（JWT + bcrypt）
- 个人信息查询（GET /api/v1/students/self）
- 合同/剩余课时查询（GET /api/v1/students/self/contracts）
- 课时记录查询（GET /api/v1/students/self/lessons）

### 核心 API 列表
POST /api/v1/auth/login
GET /api/v1/auth/me
GET /api/v1/classes?status=ACTIVE
GET /api/v1/classes/:code/students
POST /api/v1/lessons
GET /api/v1/students/self
GET /api/v1/students/self/contracts
GET /api/v1/students/self/lessons

### 测试
75 suites, 935 tests, ALL PASS（全量测试通过）

### 数据链路
user -> student（userId 关联）-> contract（studentCode 关联）-> lesson（通过 lesson_attendance 关联）

### 架构
后端：NestJS + TypeScript + TypeORM + MySQL
小程序：微信原生小程序
模块：identity/auth + teaching（class/contract/lesson/attendance）+ student

### 已知问题
1. 数据库无 migration 机制（synchronize: false + 无 migration 目录），schema 变更需手动执行 SQL
2. Student.userId 列通过 ALTER TABLE 手动添加，seed 已支持 userId:3 持久化
3. 构建有 27 个 TS 编译错误（主要在测试文件和 isolatedModules 导出不匹配），不影响运行时
4. Seed 管理员密码有 admin123 硬编码回退
5. .env 中的 DB 密码和 JWT 密钥需要生产环境独立配置
6. 微信小程序需通过开发者工具联调确认 UI/UX
7. NODE_ENV 未设置，数据库日志默认 development 模式
