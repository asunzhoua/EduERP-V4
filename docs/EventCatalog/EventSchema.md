# EduOS Event Schema Catalog

> **Version**: v1.0.0
> **Last Updated**: 2026-07-14
> **Sprint**: 4.1.6 (Governance Assets)
> **Purpose**: Technical contract for every domain event payload. Implementers must conform to these schemas when publishing or consuming events.

---

## Schema Conventions

### Standard Envelope

Every event payload MUST include these envelope fields at the top level:

```json
{
  "eventId": "string (UUID v4)",
  "timestamp": "string (ISO-8601)",
  "version": "string (semver, e.g. '1.0.0')"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `eventId` | UUID v4 | Yes | Unique event identifier. Generated by EventBusService. |
| `timestamp` | ISO-8601 | Yes | When the event was emitted. Generated by EventBusService. |
| `version` | semver | Yes | Schema version. Bumped on breaking changes. |

### Naming Rules

- Event names: `<domain>.<action>[.<context>]` (lowercase dot notation)
- Field names: camelCase
- Timestamps: ISO-8601 UTC
- IDs: Use entity primary key type (number for TypeORM bigint, string for codes)

### Versioning

- **Patch bump** (1.0.x): Adding optional fields, backward compatible
- **Minor bump** (1.x.0): Adding required fields with defaults, or new event variants
- **Major bump** (x.0.0): Removing fields, changing types, renaming fields

### Delivery Guarantees

All events use **at-least-once** delivery via `@nestjs/event-emitter`. Consumers MUST be idempotent.

### `lesson.completed`

| | |
|---|---|
| **Owner** | Teaching Domain |
| **Version** | 1.0.0 |
| **Trigger** | Lesson status: TEACHING -> FINISHED |
| **Status** | CURRENT |

**Payload:**

```json
{
  "eventId": "uuid",
  "timestamp": "2026-07-14T10:30:00Z",
  "version": "1.0.0",
  "lessonId": 12345,
  "classCode": "CLS-2026-001",
  "courseCode": "CRS-MATH-101",
  "teacherId": 1001,
  "scheduledDate": "2026-07-14",
  "actualStartTime": "2026-07-14T09:00:00Z",
  "actualEndTime": "2026-07-14T10:30:00Z",
  "durationMinutes": 90,
  "attendance": [
    {
      "studentCode": "STU-001",
      "status": "PRESENT"
    },
    {
      "studentCode": "STU-002",
      "status": "LATE"
    }
  ]
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `lessonId` | number | Yes | Lesson primary key |
| `classCode` | string | Yes | Class code (e.g., CLS-2026-001) |
| `courseCode` | string | Yes | Course code (e.g., CRS-MATH-101) |
| `teacherId` | number | Yes | Teacher user ID |
| `scheduledDate` | string | Yes | Scheduled date (YYYY-MM-DD) |
| `actualStartTime` | ISO-8601 | Yes | Actual lesson start time |
| `actualEndTime` | ISO-8601 | Yes | Actual lesson end time |
| `durationMinutes` | number | Yes | Lesson duration in minutes |
| `attendance` | array | Yes | Array of attendance records |
| `attendance[].studentCode` | string | Yes | Student code |
| `attendance[].status` | enum | Yes | One of: PRESENT, ABSENT, LATE, LEAVE, MAKEUP, ONLINE, OFFLINE |

**Consumers:** Dashboard (real-time stats), Notification (future)
**Idempotency Key:** `lessonId`
