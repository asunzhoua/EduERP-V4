## 9. Section 7: Known Gaps and Technical Debt

### 9.1 Attendance Field Gap (Architecture Decision)

| | |
|---|---|
| **Gap** | `attendance[]` field is defined in EventSchema for both `lesson.completed` and `lesson.finished`, but omitted from the actual publish payload |
| **Reason** | Attendance tracking is cross-domain. LessonService does not have access to attendance data. The Attendance subdomain will populate this field when implemented. |
| **Severity** | P2 (architecture decision, not a bug) |
| **Resolution** | When Attendance subdomain is implemented, add `attendance[]` to the publish payload |
| **Tracking** | GF-002, GF-003 (automation can validate field completeness) |

### 9.2 Event Class Dead Code (GF-002)

| | |
|---|---|
| **Gap** | `LessonCompletedEvent` and `LessonFinishedEvent` classes exist but are never instantiated in the publish flow |
| **Reason** | `EventBusService.publish()` takes a plain object, not an event class instance |
| **Severity** | P2 (dead code, no runtime impact) |
| **Resolution** | Sprint 4.2 automation: either generate event classes from EventSchema, or enforce that publish() accepts event class instances |
| **Tracking** | GF-002 |
