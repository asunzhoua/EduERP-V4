## 一、设计目标

EduOS 微信小程序不是展示系统。

而是一个每天都会高频使用的工具。

因此必须满足以下目标：

1. 页面简单
2. 操作路径最短
3. 数据实时同步
4. 所有业务计算均由服务器完成
5. AI不得自行设计页面结构
6. 页面风格统一
7. 所有端体验保持一致

---

## 二、整体页面原则（UI Constitution）

整个系统遵守以下原则：

### 原则一：一个页面只做一件事情

例如：

- 课时页面 → 只看课时
- 请假页面 → 只申请请假
- 课程反馈 → 只查看反馈

禁止一个页面承担多个业务。

### 原则二：首页只展示最重要的数据

首页不允许放：

- 宣传图
- 轮播图
- 广告
- AI助手入口
- 新闻

首页必须直接进入业务。

### 原则三：所有业务计算来自服务器

小程序：

禁止计算：

- 剩余课时
- 工资
- 积分
- 财务
- 收费

服务器返回什么就显示什么。

### 原则四：按钮越少越好

每个页面：

建议：

≤4个主要按钮

禁止：

十几个按钮同时出现。

### 原则五：所有页面风格一致

统一：

- 顶部标题
- 卡片布局
- 圆角
- 间距
- 按钮颜色
- 图标风格

禁止不同页面不同风格。

