## 2.1 微信登录能力

### 检查项

**当前登录流程：**
- 使用用户名 + 密码登录（username/password）
- 前端：`pages/login/login.js` → `post('/auth/login', { username, password })`
- 后端：`auth.controller.ts` → `AuthService.validateUser()` → bcrypt 校验
- JWT Token 机制：accessToken（2h）+ refreshToken（7d）
- Token 存储：`wx.setStorageSync('token', data.accessToken)`

**wx.login() 调用位置：**
- ❌ 未调用。前端无 `wx.login()` 调用

**code 换取 session_key 的逻辑：**
- ❌ 未实现。后端无 `sns/jscode2session` 接口调用

**用户身份绑定（微信 openid ↔ 系统 userId）：**
- ⚠️ 数据库层面已预留字段
- User entity 包含 `openid: varchar(100, nullable)` 和 `unionid: varchar(100, nullable)`
- 但无业务逻辑将 openid 与 userId 绑定

### 现状评估

⚠️ 部分实现

- ✅ JWT 认证体系完整（accessToken + refreshToken + login log）
- ✅ User 表已预留 openid / unionid 字段
- ❌ 无 wx.login() 调用
- ❌ 无 code → session_key → openid 流程
- ❌ 无微信登录 API 端点（如 `/auth/wx-login`）
- ❌ 前端 `onWechatLogin()` 仅为 placeholder（显示"微信授权登录待实现"）

### 改进建议

需要实现完整的微信登录流程：

1. **前端**：
   - 调用 `wx.login()` 获取 code
   - 将 code 发送到后端 `/auth/wx-login`
   - 处理首次登录（需绑定手机号或创建账号）和已绑定登录两种场景

2. **后端**：
   - 新增 `/auth/wx-login` 端点（Public）
   - 调用微信 `sns/jscode2session` API（需 AppID + AppSecret）
   - 用 openid 查找 User，若存在则签发 JWT
   - 若不存在则返回"需绑定"状态，引导用户绑定手机号

3. **配置需求**：
   - 微信小程序 AppID
   - 微信小程序 AppSecret
   - 后端环境变量：`WX_APPID`、`WX_APP_SECRET`

**订阅消息模板申请状态：**

**模板 ID 配置：**
- ❌ 未配置。无模板 ID 常量或环境变量

**发送接口实现：**
- ❌ 未实现。后端无订阅消息发送逻辑

**用户授权流程：**
- ❌ 未实现。前端无 `wx.requestSubscribeMessage()` 调用

### 现状评估

❌ 未实现

- 无任何订阅消息相关代码
- 无模板 ID 配置
- 无发送接口
- 无用户授权流程

### 改进建议

需要从零实现订阅消息能力：

1. **微信公众平台配置**：
   - 登录小程序后台 → 订阅消息 → 选择模板
   - 推荐模板场景：
     - 上课提醒（课程名称 + 上课时间 + 教室）
     - 考勤通知（到课/缺课 + 课程名称 + 时间）
     - 课时变动（剩余课时 + 课程名称）
     - 作业提醒（作业内容 + 截止时间）

2. **前端实现**：
   - 在关键操作后调用 `wx.requestSubscribeMessage()` 请求用户授权
   - 时机建议：登录成功后、课时完成后、作业布置后
   - 模板 ID 配置在 `config.js` 中

3. **后端实现**：
   - 封装微信订阅消息发送服务
   - 调用 `POST https://api.weixin.qq.com/cgi-bin/message/subscribe/send`
   - 需要 access_token（通过 AppID + AppSecret 获取）
   - 在业务事件触发时发送（如考勤确认、课时完成）

4. **数据库扩展**：
   - User 表需确保 openid 已绑定（订阅消息需要 openid）
   - 可选：新增订阅消息发送记录表

5. **需要的配置**：
   - 小程序 AppID + AppSecret（获取 access_token）
   - 订阅消息模板 ID（至少 2-3 个核心场景）

## 2.3 微信用户绑定能力

### 检查项

**微信用户信息获取（头像、昵称）：**
- ❌ 未实现。前端无 `wx.getUserProfile()` 或头像昵称填写组件调用
- 注：微信 2022 年后已废弃 `wx.getUserProfile()`，需使用头像昵称填写组件（`<button open-type="chooseAvatar">` + nickname input）

**用户信息存储（数据库字段）：**
- ⚠️ 部分预留
- User entity 有 `avatar: varchar(255, nullable)` — ✅ 可存储头像 URL
- User entity 有 `name: varchar(50)` — ✅ 可存储昵称（但当前用于系统姓名）
- ❌ 无独立的微信昵称字段（nickname）

**用户信息展示（小程序页面）：**
- ❌ 未实现。首页 `index.js` 读取 `app.globalData.userInfo` 但仅展示角色文本
- 无头像展示组件
- 无个人中心页面

### 现状评估

⚠️ 部分实现

- ✅ User 表有 avatar 字段
- ✅ User 表有 name 字段
- ❌ 无微信头像/昵称获取逻辑
- ❌ 无个人中心页面
- ❌ 无头像展示 UI

### 改进建议

1. **前端**：
   - 新增个人中心页面（`pages/profile/profile`）
   - 使用微信头像填写组件 `<button open-type="chooseAvatar">` 获取头像
   - 使用 nickname input 组件获取昵称
   - 头像上传到后端或直接使用微信临时 URL

2. **后端**：
   - 新增 `/user/profile` PUT 端点（更新 avatar、nickname）
   - 可选：头像图片上传接口（存 OSS 或本地）

3. **数据库**：
   - 可选：新增 `wechatNickname` 字段（与系统 name 区分）

## 2.8 实现优先级

### P0 — 微信登录（阻塞项）

**原因：**
- 当前用户名+密码登录对 C 端用户（家长/学生）不友好
- 微信登录是小程序的基础能力
- 数据库已预留 openid/unionid 字段，改造成本可控

**预估工作量：** 2-3 天
**依赖：** 需要 Owner 提供 AppID + AppSecret

### P1 — 订阅消息（核心运营能力）

**原因：**
- 上课提醒、考勤通知是教培核心场景
- 替代人工通知，提升运营效率
- 用户粘性的重要手段

**预估工作量：** 3-5 天
**依赖：** 需要 Owner 申请订阅消息模板 + 配置 AppID/AppSecret

### P2 — 用户绑定（体验优化）

**原因：**
- 头像/昵称展示提升用户体验
- 个人中心是小程序标配页面
- 非核心功能，不影响业务流程

**预估工作量：** 1-2 天
**依赖：** 无外部依赖

### P3 — 分享功能（运营推广）

**原因：**
- 分享是获客手段
- 非核心功能，可后续迭代
- 需要配合运营策略

**预估工作量：** 1-2 天
**依赖：** 需要设计分享卡片图片

## 总结

### 能力矩阵

| 能力 | 状态 | 优先级 | 阻塞项 |
|:-----|:-----|:------|:------|
| 微信登录 | ⚠️ 部分实现 | P0 | 需 AppID + AppSecret |
| 订阅消息 | ❌ 未实现 | P1 | 需模板申请 + AppID |
| 用户绑定 | ⚠️ 部分实现 | P2 | 无 |
| 微信支付 | 🚫 暂不需要 | — | — |
| 分享功能 | ❌ 未实现 | P3 | 需设计素材 |
| 微信卡券 | 🚫 暂不需要 | — | — |

### 关键发现

1. **数据库层面准备度较好**：User 表已预留 openid、unionid、avatar 字段，说明架构设计时已考虑微信生态集成
2. **后端认证体系完整**：JWT + refreshToken 机制成熟，微信登录只需新增一个端点，复用现有 JWT 签发逻辑
3. **前端登录页已预留入口**：`onWechatLogin()` 方法和"微信授权登录"按钮已存在，只需实现逻辑
4. **最大阻塞项**：AppID + AppSecret 配置（需 Owner 在微信公众平台获取）

### 建议执行顺序

```
Step 1: Owner 配置 AppID + AppSecret → 环境变量
Step 2: 实现微信登录（P0）— 2-3 天
Step 3: 实现订阅消息（P1）— 3-5 天
Step 4: 实现用户绑定 + 个人中心（P2）— 1-2 天
Step 5: 实现分享功能（P3）— 1-2 天
```

## Decision Gate

⚠️ **以下配置需要 Owner 手动完成，AI 无法自动配置：**

1. **微信小程序注册**（如未注册）
