## Summary

## What's New

### Authentication
- POST /api/v1/auth/login — BCrypt password hashing, JWT access token (2h)
- POST /api/v1/auth/logout — Clear refresh token, record login log
- POST /api/v1/auth/refresh — Token rotation, refresh token valid 7d (DB-stored)
- GET /api/v1/auth/me — Current user profile (sensitive fields excluded)

### Database Tables (6)
- `user` — Users table with BCrypt passwords + refresh token storage
- `role` — Role definitions (SuperAdmin, Admin, Teacher, Parent)
- `permission` — Permission codes (12 initial permissions across 6 modules)
- `user_role` — User-role many-to-many relations
- `role_permission` — Role-permission many-to-many relations
- `login_log` — Login/logout/refresh audit trail

### Security
- JwtAuthGuard registered globally (all routes protected by default)
- @Public() decorator for public endpoints (login, refresh)
- RolesGuard + @Roles() decorator for RBAC permission checks
- Refresh tokens stored in DB (supports admin kick-out / forced re-login)
- BCrypt password hashing (salt rounds: 10)
