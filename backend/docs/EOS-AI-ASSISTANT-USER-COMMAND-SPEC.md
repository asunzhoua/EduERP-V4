## 1. 命令模型

### 1.1 命令格式

所有命令通过 **@EOS AI Assistant** 触发：

```
@EOS AI Assistant <动词> [参数]
```

| 部分 | 说明 | 示例 |
|:-----|:------|:------|
| `@EOS AI Assistant` | 触发的必选前缀 | `@EOS AI Assistant` |
| `<动词>` | 操作类型，位置固定 | `状态`, `创建`, `暂停` |
| `[参数]` | 操作对象或附加信息 | `M-2026-07-18-001` |

完整示例：
```
@EOS AI Assistant 查看 M-2026-07-18-001
@EOS AI Assistant 列表 已完成
@EOS AI Assistant 创建 扫描所有 TypeScript 错误
```

### 1.2 设计原则

| 原则 | 说明 |
|:-----|:------|
| **自然语言优先** | 命令格式宽松，GPT 负责解析意图 |
| **容错** | 支持近义词：`查看` / `详情` / `查` / `show` |
| **渐进式披露** | 初次交互只展示基础命令，高级命令需要 help 或经验积累 |
| **反馈明确** | 每个命令必须给出明确的状态反馈 |
| **错误友好** | 不理解时给出建议，而非报错 |

### 1.3 对话状态管理

GPT 维护对话状态，支持多轮交互：

```
用户: @EOS AI Assistant 创建 扫描所有 TS 错误
AI:   📝 已生成 Draft，请确认:
      T001: 扫描项目中的 TS 错误
      T002: 分析错误类型
      ─────
      回复 "确认" 以创建任务

用户: 修改，增加"修复"步骤
AI:   ✅ 已更新：
      T001: 扫描项目中的 TS 错误
      T002: 分析错误类型
      T003: 修复所有 TS 错误
      ─────
      回复 "确认" 以创建任务

用户: 确认
AI:   ✅ Mission M-2026-07-18-003 已创建！
```

对话状态在 GPT 会话上下文中维护，超时 5 分钟后自动清除。

### 2.2 创建类命令

#### CMD-05: 创建 Mission Draft

| 属性 | 值 |
|:-----|:----|
| **动词** | `创建`, `create`, `创建任务` |
| **参数** | 任务描述（可选，长描述可分多轮对话提供） |
| **权限** | 主人 + 授权协作者 |
| **对应 API** | `lobster-api --draft create <DESCRIPTION>` |

**响应模板**：
```
📝 已生成 Mission Draft
═══════════════════════════════
🎯 任务名: 修复 TypeScript 编译错误

📋 任务分解 (3 steps):
  ├─ T001: 扫描所有 TS 错误
  │   └─ 扫描项目 src/ 目录，列出所有 TypeScript 编译错误
  ├─ T002: 修复 TS 错误
  │   └─ 逐一修复 T001 发现的错误
  └─ T003: 验证修复
      └─ 运行 tsc --noEmit 确认无错误

────────────────────────────────
💡 你可以:
  • 回复 "确认" 来创建这个任务
  • 回复 "修改" 来调整任务
  • 回复 "取消" 来放弃
```

**多轮交互 — 修改 Draft**：
```
用户: 增加一个测试步骤
AI:   ✅ 已更新 Draft:
      T001: 扫描所有 TS 错误
      T002: 修复 TS 错误
      T003: 运行测试
      T004: 验证无编译错误

      回复 "确认" 来创建，或继续提出修改意见

用户: 确认
AI:   ✅ Mission M-2026-07-18-004 已创建！
      状态: 📝 CREATED
      等待 Pump Runner 调度执行。
```

### 2.5 错误响应

#### ERR-01: 无法理解

```
🤔 抱歉，我不太理解您的意思。
您可以试试:
  • "状态" — 查看系统状态
  • "列表" — 查看所有任务
  • "创建 <描述>" — 创建新任务
  • "帮助" — 查看全部命令
```

#### ERR-02: 需要更多信息

```
📝 请提供更多信息:
  • 创建任务: 请描述您想完成什么，例如:
    "创建 修复 TypeScript 编译错误"
  • 查看任务: 请提供 Mission ID，例如:
    "查看 M-2026-07-18-001"
```

#### ERR-03: 无权限

```
⛔ 抱歉，此操作需要管理员权限。
如果您是主人，请输入管理员口令继续。
```

#### ERR-04: 找不到任务

```
🔍 未找到 Mission: M-2026-07-18-999
请检查 Mission ID 是否正确，或输入 "列表" 查看所有任务。
```

#### ERR-05: 状态不匹配

```
⛔ 无法暂停 M-2026-07-18-001
当前状态: COMPLETED
只能暂停 RUNNING 状态的 Mission。
```

#### ERR-06: 系统错误

```
🚨 请求处理失败
原因: Lobster API 连接超时
建议: 请稍后重试，或检查系统状态
```

## 3. 命令解析规则

### 3.1 GPT 解析优先级

```
收到用户消息
    │
    ├─ 是否是对话上下文延续（前一轮未完成）？
    │   └─ 是 → 检查对话状态
    │       ├─ 等待确认 → 处理 "确认"/"修改"/"取消"/"调整"
    │       ├─ 等待口令 → 检查口令
    │       └─ 等待补充 → 追加到当前 Draft
    │
    ├─ 是否包含命令动词？
    │   ├─ "状态"/"status" → CMD-01
    │   ├─ "健康"/"health" → CMD-02
    │   ├─ "列表"/"list" → CMD-03
    │   ├─ "查看"/"详情"/"show" → CMD-04
    │   ├─ "创建"/"create" → CMD-05
    │   ├─ "快速创建"/"quick" → CMD-06
    │   ├─ "暂停"/"pause" → CMD-07
    │   ├─ "恢复"/"resume" → CMD-08
    │   ├─ "中止"/"abort" → CMD-09
    │   └─ "帮助"/"help" → CMD-10
    │
    ├─ 包含 Mission ID 格式 (M-YYYY-MM-DD-NNN)？
    │   └─ 是 → 推断意图：查看/操作
    │
    └─ 全部不匹配 → ERR-01: 无法理解
```

### 3.2 参数提取规则

```
"查看 M-2026-07-18-001"
  → 动词: 查看
  → mission_id: M-2026-07-18-001

"列表 已完成"
  → 动词: 列表
  → filter: COMPLETED

"创建 修复数据库连接超时，前端显示报错"
  → 动词: 创建
  → description: "修复数据库连接超时，前端显示报错"

"暂停 M-2026-07-18-001"
  → 动词: 暂停
  → mission_id: M-2026-07-18-001
  → 需要: 口令验证
```

### 3.3 同义词映射

| 命令 | 同义词（均支持） |
|:-----|:----------------|
| `状态` | `status`, `概览`, `系统状态`, `系统概况` |
| `健康` | `health`, `检查`, `体检`, `系统健康` |
| `列表` | `list`, `任务`, `missions`, `任务列表`, `全部任务` |
| `查看` | `详情`, `show`, `info`, `detail`, `任务详情` |
| `创建` | `create`, `新建`, `创建任务`, `新任务` |
| `暂停` | `pause`, `stop`, `停止`, `挂起` |
| `恢复` | `resume`, `continue`, `继续`, `重启` |
| `中止` | `abort`, `取消任务`, `终止` |
| `帮助` | `help`, `?`, `命令`, `使用说明` |

### 3.4 Mission ID 格式规范

| 格式 | 示例 | 说明 |
|:-----|:------|:------|
| 完整 ID | `M-2026-07-18-001` | 标准格式，由 Lobster 生成 |
| 缩写（对话上下文） | `001` | 只有在对话历史中提及对应任务时可简写 |

## 5. 渐进式披露

### 5.1 新用户初次交互

只展示最基础的命令：

```
🤖 EOS AI Assistant 已上线！

基础命令:
  "@EOS AI Assistant 状态" — 查看系统状态
  "@EOS AI Assistant 帮助" — 查看全部命令

更多功能请输入 "帮助" 查看。
```

### 5.2 完整帮助

通过 `帮助` 命令展示全部命令（参见 CMD-10）。

### 5.3 命令建议

GPT 在不确定时可主动提供建议：

```
用户: 我想看看系统怎么样
AI:   好的！以下是可能匹配的命令：
      1. "状态" — 系统整体概览
      2. "健康" — 组件健康检查
      3. "列表" — 任务列表
      请问您想看哪个？
```
